# 라디오 만드는 밤 — 소모임 사이트 설계

작성일: 2026-07-01

## 한 줄 요약

라디오 만들기 소모임이 **회차마다 주제를 정하고, 음악을 추천하고, 서로 논의하는** 감성 프로토타입 웹사이트. 백엔드 없이 브라우저 안에서 실제처럼 동작한다.

## 목표와 범위

- **목표**: 실제 서비스처럼 보이고 만져지는, 감성 넘치고 직관적인 프로토타입.
- **성격**: 감성 프로토타입 (데이터 영속화·로그인 없음). 상태는 `localStorage`에 저장해 새로고침해도 유지되되, 브라우저 안에서만 유지된다.
- **비범위(YAGNI)**: 서버/DB, 로그인/인증, 실제 음원 재생, 반응형 이외의 다국어·접근성 고도화. 나중에 "진짜 앱"으로 확장할 때 이 화면 위에 Supabase 등을 얹는다.

## 비주얼 방향 — 에디토리얼 모노

Behance "Morning aesthetics"의 따뜻함을 기준으로 하되, **극도로 깔끔·미니멀**하게. 여백이 주인공.

- **팔레트**: 배경 `#FCFBF9`, 본문 `#121212`, 흐린 텍스트 `#9A9A96`, 헤어라인 `#E7E7E4`, 포인트(따뜻한 점 하나) `#C7724C`. 색은 최소한으로.
- **타이포**:
  - 디스플레이(제목) = **GraceSerif 2.0** — 프로젝트에 폰트 파일을 포함(self-host)해 `@font-face`로 사용. 원본: `/Users/teamsparta/Documents/AX-workflow/GraceSerif2.0/GraceSerif2.0-{Regular,Bold}.otf` → `assets/fonts/`로 복사.
  - 본문·UI = **Pretendard** (웹폰트 CDN).
  - 제목은 크고 우아하게(GraceSerif), 본문은 담백하게(Pretendard).
- **디테일**: 얇은 헤어라인 구분선, 넉넉한 패딩, 군더더기 장식 없음. `● On Air · Vol.N` 같은 작은 라벨로 라디오 정체성만 은은하게.

## 구조 — 회차/에피소드 중심

각 회차(Vol.N)가 하나의 라디오 주제를 담고, 그 안에 추천곡과 논의가 모인다. 스토리가 쌓이는 아카이브.

## 화면 구성 (3개 페이지, 빌드 없는 정적 사이트)

### 1. 홈 — 온에어 데스크 (`index.html`)
- 얇은 상단 네비: 로고(라디오 만드는 밤) · 홈 · 다음 주제 정하기 · 지난 회차
- `● On Air · Vol.12` 라벨 + 이번 회차 대형 세리프 제목
- 메타 영역: 좌측 "이번 주제" 소개글, 우측 "추천곡" 요약 리스트(곡 — 추천인)
- 지난 회차 아카이브: 미니멀한 목록/스트립, 클릭 시 회차 상세로 이동
- 하단: "다음 주제 정하는 중" 티저 → 투표 페이지로 유도

### 2. 회차 상세 (`episode.html?vol=12`)
- 주제 헤더: 대형 세리프 제목 + 소개글
- 추천곡 리스트: 곡명 · 추천인 · 한 줄 사유
- **곡 추천 등록 폼**: [이 회차에 곡 추천하기] → 곡명/추천인/사유 입력 시 리스트에 즉시 추가 (localStorage 반영)
- 논의(댓글) 섹션: 닉네임 + 내용 입력 → 목록에 즉시 추가

### 3. 다음 주제 정하기 (`topics.html`)
- 제안된 주제 카드 목록 + 하트(투표) 버튼 → 누르면 카운트 상승, 인기순 정렬
- **새 주제 제안 폼**: 제안자/주제/한 줄 설명 입력 → 목록에 추가

## 데이터 모델 (localStorage)

- `episodes`: `[{ vol, title, intro, coverColor, tracks:[{title, by, reason}], comments:[{name, text, at}] }]`
- `topics`: `[{ id, title, desc, by, votes }]`
- 최초 로드 시 시드 데이터(감성 있는 샘플 회차·곡·댓글·주제)를 심어 실제처럼 보이게 한다. 사용자가 추가/투표한 내용은 localStorage에 병합 저장.

## 컴포넌트/파일 구성

빌드 없는 정적 사이트. 프레임워크·설치 없음, 파일을 열거나 간단한 정적 서버로 실행.

- `index.html`, `episode.html`, `topics.html` — 3개 페이지
- `assets/fonts/` — GraceSerif 2.0 폰트 파일(.otf) + `@font-face` 정의
- `assets/style.css` — 공통 스타일(팔레트·타이포·네비·폼·카드), 페이지별 최소 추가 스타일
- `assets/data.js` — 시드 데이터 + localStorage 읽기/쓰기 헬퍼(load/save, 병합)
- `assets/app.js` — 페이지별 렌더링·폼 제출·투표 핸들러 (경계 분리: 데이터 접근은 data.js, DOM 렌더는 app.js)
- 반응형: 모바일에서 hero 세로 스택, 폰트 축소.

## 인터랙션(살아있는 부분)

1. **댓글 논의** — 회차 상세에서 제출 시 목록 상단(또는 하단)에 즉시 추가 + 저장.
2. **곡 추천 등록** — 폼 제출 시 추천곡 리스트에 즉시 추가 + 저장.
3. **다음 주제 제안·투표** — 하트 클릭 시 카운트 증가·재정렬, 새 주제 제안 폼.
   - (재생바는 범위 제외 — 가짜 재생 대신 실제 활동에 집중)

## 검증(테스트) 방법

프로토타입이므로 자동화 테스트는 최소화하고 수동 확인 위주:
- 세 페이지가 시드 데이터로 올바르게 렌더되는가
- 댓글/곡 추천/주제 제안 제출 후 화면에 즉시 반영되고, 새로고침 후에도 유지되는가
- 투표 클릭 시 카운트·정렬이 갱신되는가
- 페이지 간 네비게이션(홈 ↔ 회차 상세 ↔ 다음 주제)이 동작하는가
- 모바일 폭에서 레이아웃이 깨지지 않는가

## 열기/공유

`index.html`을 브라우저로 직접 열거나, 로컬 정적 서버(예: `python3 -m http.server`)로 실행해 공유.
